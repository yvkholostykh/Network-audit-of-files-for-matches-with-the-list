# FileAuditor Pro – поиск конфиденциальных файлов

![Python Version](https://img.shields.io/badge/python-3.8%2B-blue)
![License](https://img.shields.io/badge/license-MIT-green)

**FileAuditor Pro** – это инструмент командной строки для сканирования файловых систем (локальных и сетевых) с целью обнаружения файлов, содержащих ключевые слова, связанные с персональными данными или чувствительной информацией. Скрипт анализирует имена файлов, а также содержимое текстовых документов (`.txt`, `.csv`, `.log`, `.xml`, `.json`, `.py`, `.md` и др.) и формирует детальный отчёт в форматах CSV, TXT или PDF.

## 📌 Возможности

- Рекурсивный обход папок с поддержкой больших каталогов (сетевые пути, миллионы файлов).
- Поиск совпадений по списку ключевых слов (загружается из внешнего файла или используется встроенный список).
- Анализ:
  - имени файла;
  - содержимого текстовых файлов (первые 1 МБ текста для производительности).
- Определение владельца файла (на Windows через `pywin32`).
- Прогресс-бары при наличии библиотек `tqdm` и `alive-progress`.
- Вывод результатов в консоль с цветовым выделением (при наличии `colorama`).
- Сохранение отчётов в трёх форматах:
  - **CSV** – для дальнейшего анализа в Excel или Pandas.
  - **TXT** – читаемый человеком отчёт с деталями.
  - **PDF** – красивый форматированный документ (требуется `fpdf2` и шрифт с кириллицей).

## 🛠 Требования

- **Python 3.8+**
- **Операционная система:** Windows (из-за использования `pywin32` для получения владельца; при необходимости можно адаптировать под Linux/macOS, но текущая версия заточена под Windows).
- **Дополнительные библиотеки** (стандартные): `os`, `sys`, `getpass`, `threading`, `csv`, `json`, `datetime`, `pathlib`, `argparse`.

### Необязательные (но рекомендуемые) зависимости

| Библиотека         | Назначение                                        | Как установить                  |
|--------------------|---------------------------------------------------|---------------------------------|
| `colorama`         | Цветной вывод в консоль                          | `pip install colorama`          |
| `tqdm`             | Красивый прогресс-бар для сканирования           | `pip install tqdm`              |
| `alive-progress`   | Анимированный прогресс-бар (дополнительно)       | `pip install alive-progress`    |

> 💡 **Важно:** код умеет работать с **локальными копиями** библиотек, если разложить их в папку `C:/Users/.../FileAuditor` (см. переменную `LIB_PATHS`). Однако для общего использования проще установить зависимости через `pip`.

## 📥 Установка и подготовка

### 1. Скачайте скрипт

Создайте файл `fileauditor.py` и скопируйте в него приведённый выше код.

### 2. Установите зависимости (рекомендуемый способ)

```bash
pip install colorama tqdm alive-progress progress pywin32 fpdf2

Если вы не планируете использовать PDF или определение владельца, соответствующие библиотеки можно не устанавливать – скрипт сам предупредит об отсутствии.

3. Подготовьте файл с ключевыми словами (опционально)
Создайте текстовый файл (например, keywords.txt), в котором каждое ключевое слово или фраза находятся на отдельной строке. Пустые строки и строки, начинающиеся с #, игнорируются.

Пример keywords.txt:
паспорт
снилс
инн
договор закупки
конфиденциально
коммерческая тайна
...
Если файл не указан, скрипт использует встроенный стандартный список:
пдн, паспорт, снилс, инн, договор, закупка, контракт, билеты, медкнижка, диплом, удостоверение, фио, фамилия, имя, отчество.

4. Подготовьте шрифт для PDF (для корректной кириллицы)
По умолчанию скрипт ищет шрифт C:\Windows\Fonts\DejaVuSans.ttf. Если его нет, PDF будет использовать стандартный Arial, который может не отображать русские буквы.
Рекомендация: скачайте DejaVuSans.ttf (или любой другой шрифт с поддержкой Unicode) и поместите в C:\Windows\Fonts\ или измените путь в коде (переменная font_path в методе save_report_pdf).

🚀 Использование
Синтаксис командной строки
python fileauditor.py [папка_сканирования] [-k файл_ключевых_слов]
Аргументы:

папка_сканирования – путь к директории, которую нужно просканировать. Если не указан, программа запросит его интерактивно.

-k, --keywords – путь к текстовому файлу со списком ключевых слов (построчно). Если не указан, программа спросит, хотите ли вы его задать, или использует стандартный список.

Интерактивный режим
Если запустить скрипт без аргументов, он запросит:

Путь к папке для сканирования.

Путь к файлу ключевых слов (можно пропустить Enter, тогда будут использованы стандартные слова).

После завершения сканирования будет предложено сохранить отчёт в одном из доступных форматов (CSV, TXT, PDF). Выбор формата и пути сохранения – по желанию.

Примеры
Пример 1: сканирование с ключевыми словами из файла, папка указана в аргументе.
python fileauditor.py D:\Документы -k my_keywords.txt
Пример 2: сканирование текущей папки без указания ключевых слов (будут стандартные).
python fileauditor.py .
Пример 3: полный интерактивный запуск.
python fileauditor.py
📄 Форматы отчётов
CSV
Файл с разделителями-запятыми (UTF-8 с BOM), содержит колонки:
path, filename, matched_keywords, match_type, owner, size, modified.

TXT
Читаемый текстовый отчёт с заголовком, датой, общим числом совпадений и детализацией по каждому файлу (путь, владелец, размер, дата изменения, ключевые слова).

PDF
Многостраничный документ (при большом количестве записей вывод ограничен 100 для компактности). Кириллица корректно отображается при наличии шрифта DejaVuSans.ttf.

⚙️ Особенности работы кода
Производительность: для больших папок (десятки/сотни тысяч файлов) рекомендуется установить tqdm и alive-progress – они улучшат визуальную обратную связь. Без них используется простой текстовый индикатор каждые 100 файлов.

Обработка ошибок: скрипт не прерывается при невозможности прочитать файл или получить его владельца. Все ошибки собираются в список и выводятся в конце.

Память: содержимое файлов читается лимитированно (первые 1 МБ), что позволяет быстро обрабатывать даже очень большие текстовые файлы.

Кодировка: для чтения файлов используется utf-8 с игнорированием ошибок (errors='ignore'). Это может пропустить некоторые символы, но решает проблему со смешанными кодировками.

❓ Часто задаваемые вопросы
Q: Почему владелец файла отображается как "N/A"?
A: Скорее всего, не установлена библиотека pywin32 или скрипт запущен на Linux/macOS. Для Windows выполните pip install pywin32.

Q: PDF создаётся, но русский текст не отображается.
A: Установите шрифт с поддержкой кириллицы. Проще всего скачать DejaVuSans.ttf и положить в C:\Windows\Fonts\. Либо измените путь в коде на любой другой .ttf-файл.

Q: Сканирование очень медленное на сетевой папке.
A: Это нормально для сетевых дисков. Для ускорения можно исключить файлы большого размера, изменив код (например, добавить фильтр по расширению). Также прогресс-бары могут немного замедлять работу – их можно отключить, удалив соответствующие импорты.

Q: Как добавить новые расширения для анализа содержимого?
A: В классе FileAuditorPro найдите атрибут TEXT_EXTENSIONS (множество строк) и добавьте нужные расширения, например '.docx' (но учтите, что это бинарный формат – содержимое читаться не будет). Для реального анализа .docx потребуется дополнительная библиотека (python-docx).

Q: Можно ли использовать скрипт на сервере без графического интерфейса?
A: Да, полностью. Скрипт не требует GUI, все операции выполняются через консоль. Прогресс-бары работают в текстовом режиме.

🐛 Устранение неполадок

Проблема	Возможное решение
ImportError: No module named 'colorama'	Установите: pip install colorama или удалите зависимость из кода, удалив строки импорта и проверки.
PermissionError при доступе к файлу	Запустите скрипт с правами администратора или убедитесь, что файлы не заблокированы другим процессом.
LookupAccountSid ошибка при получении владельца	Убедитесь, что путь существует и у вас есть права на чтение SECURITY_DESCRIPTOR.
Прогресс-бары не отображаются	Установите недостающие библиотеки (tqdm, alive-progress). Либо это нормально – скрипт переключится на упрощённый вывод.

## 📖 Инструкция по использованию кода (краткое руководство)

1. **Сохраните скрипт** в файл `fileauditor.py`.
2. **Установите зависимости** через `pip` (см. выше).
3. **Создайте файл ключевых слов** (опционально) в любом текстовом редакторе, одно слово/фраза на строку.
4. **Запустите сканирование**:
   - Откройте командную строку (cmd, PowerShell или терминал).
   - Перейдите в папку со скриптом: `cd C:\путь\к\вашей\папке`
   - Выполните:  
     `python fileauditor.py "D:\Мои документы" -k keywords.txt`
   - Или запустите без параметров и отвечайте на вопросы.
5. **Дождитесь** завершения сканирования (прогресс-бары показывают состояние).
6. **Просмотрите найденные совпадения** – они выводятся в консоль (первые 20).
7. **Сохраните отчёт**, выбрав формат (`csv`, `txt` или `pdf`) и указав путь.
8. **Откройте отчёт** в Excel, Notepad++ или любом просмотрщике PDF.
